# API TESTING FRAMEWORK - MANUAL DE USUARIO

Framework profesional para pruebas automatizadas de APIs REST
Tecnologías: Java 21, Maven, Cucumber, Rest Assured, Serenity BDD

## 1. CARACTERÍSTICAS

| Categoría       | Características                                                                 |
|-----------------|---------------------------------------------------------------------------------|
| Autenticación   | Validación de headers requeridos, API keys, timestamps, tokens JWT              |
| Autorización    | Control de acceso por roles, permisos por endpoint, validación de scopes        |
| Funcionalidad   | Operaciones CRUD, validaciones de negocio, valores dinámicos                    |
| Configuración   | Definición declarativa en YAML, sin código Java para pruebas                    |
| Pre-script      | Obtención automática de headers desde servicios externos con cache              |
| Reportes        | Markdown detallado, Cucumber HTML, Serenity BDD profesional                     |
| Dinamismo       | Generación automática de emails, UUIDs, timestamps, datos aleatorios            |
| Rendimiento     | Ejecución paralela, cache de headers, timeouts configurables                    |

## 2. REQUISITOS PREVIOS

- Java JDK 21 o superior
- Maven 3.6 o superior
- Git (opcional)

Verificación:
  java -version   # Debe mostrar Java 21
  mvn -version    # Debe mostrar Maven 3.6+

Instalación rápida:
  Ubuntu/Debian: sudo apt update && sudo apt install openjdk-21-jdk maven
  macOS: brew install openjdk@21 maven
  Windows: Descargar desde https://adoptium.net/ (Java 21) y https://maven.apache.org/download.cgi

## 4. INSTALACIÓN

Paso 1: Clonar el repositorio
  git clone https://github.com/tu-usuario/api-testing-framework.git
  cd api-testing-framework

Paso 2: Configurar variables de entorno
  Editar src/test/resources/config.env:
    BASE_URL=https://api.ejemplo.com/v2
    API_KEY=tu-api-key-aqui
    TEST_USER=usuario@ejemplo.com
    TEST_PASSWORD=tu-contraseña
    APPLICATION_NAME=mi-aplicacion
    ENVIRONMENT_TYPE=testing

Paso 3: Configurar headers base
  Editar src/test/resources/headers/base-headers.yaml:
    headers:
      base:
        X-API-Key: "${API_KEY}"
        X-Application: "${APPLICATION_NAME}"
        Content-Type: "application/json"
        Accept: "application/json"
      pre_script:
        enabled: false
        headers: {}
      dynamic:
        generate_transaction_id: true
        generate_timestamp: true
      required:
        - "X-API-Key"
        - "X-Transaction-ID"
        - "X-Timestamp"

Paso 4: Compilar el proyecto
  mvn clean compile

Paso 5: Verificar la instalación
  mvn test -Dcucumber.filter.tags="@Smoke"

Instalación con Docker (alternativa)
  Crear Dockerfile:
    FROM maven:3.9-eclipse-temurin-21
    WORKDIR /app
    COPY pom.xml .
    COPY src ./src
    RUN mvn clean compile
    CMD ["mvn", "test"]
  Construir: docker build -t api-testing-framework .
  Ejecutar: docker run --rm -v $(pwd)/target:/app/target api-testing-framework

## 5. CONFIGURACIÓN

Archivos principales:

- config.env: Variables de entorno (URLs, credenciales, timeouts, flags)
- base-headers.yaml: Headers base, pre-script dinámico, headers requeridos/opcionales
- cucumber.properties: Configuración de Cucumber (plugins, glue, tags, paralelismo)
- suites YAML: Definición de pruebas en formato declarativo

Ejemplo de suite YAML:
  suite:
    name: "Pruebas Funcionales"
    type: "functional"
    base_url: "https://api.ejemplo.com/v2"
    tests:
      - name: "Crear usuario"
        method: "POST"
        endpoint: "/api/v1/usuarios"
        body:
          content:
            nombre: "María González"
            email: "maria@ejemplo.com"
        expected:
          status: 201
        extract:
          - name: "usuario_id"
            from: "$.id"

Perfiles de configuración:
  src/test/resources/config-dev.env, config-qa.env, config-prod.env
  Uso: cp config-qa.env config.env && mvn test

Validación de configuración:
  cat src/test/resources/config.env
  yamllint src/test/resources/headers/base-headers.yaml
  mvn test -Dcucumber.execution.dry-run=true
  mvn clean test && ls -la target/reports/
